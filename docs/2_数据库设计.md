# 数据库设计方案 (PostgreSQL)

本系统采用 **PostgreSQL 15+** 作为核心数据库，充分利用其 **Schema（模式）** 机制进行模块解耦，并使用 **JSONB** 与 **Array** 类型处理半结构化数据。

---

## 1. 设计规范与约定

### 1.1 命名规范
- **表名/字段名**: 使用 `snake_case` (如 `user_id`, `created_at`)。
- **主键**: 统一命名为 `id`，类型为 `BIGSERIAL` 或 `BIGINT` (Snowflake算法)。
- **外键**: 命名为 `关联表单数_id` (如 `student_id`)。
- **通用字段**: 所有表默认包含 `created_at`, `updated_at`, `is_deleted` (逻辑删除)。

### 1.2 高级特性应用
- **Schema 隔离**: 不同微服务/模块的数据表存放在不同的 Schema 中（如 `auth`, `student`, `project`），便于权限管理和未来拆库。
- **JSONB**: 用于存储动态表单、简历详情、评价维度等非固定结构数据。
- **Array**: 用于存储简单的标签（Tags）、多选值，减少简单的 `1:N` 关联表。

## 3. 详细表结构设计

### 3.1 认证与基础中心 (Schema: `auth_center`)

负责用户账号、角色权限及租户（学校/企业）管理。

#### `sys_tenant` (租户/机构表)
| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | BIGSERIAL | 主键 |
| name | VARCHAR(100) | 机构名称 (xx大学/xx科技公司) |
| type | SMALLINT | 类型 (1:高校, 2:企业, 0:平台运营) |
| status | SMALLINT | 状态 (0:待审核, 1:正常, 2:禁用) |
| config | JSONB | 租户配置 (Logo, 主题色, 审核配置) |

#### `sys_user` (系统用户表)
| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | BIGSERIAL | 主键 |
| tenant_id | BIGINT | 所属租户ID |
| username | VARCHAR(50) | 账号 |
| password | VARCHAR(100) | 加密密码 |
| phone | VARCHAR(20) | 手机号 |
| role_ids | BIGINT[] | 角色ID列表 (Array) |
| status | SMALLINT | 状态 (1:正常, 2:锁定) |

#### `sys_role` (角色表)
| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | BIGSERIAL | 主键 |
| tenant_id | BIGINT | 归属租户 (0代表系统通用角色) |
| code | VARCHAR(50) | 角色编码 (ROLE_STUDENT, ROLE_HR) |
| name | VARCHAR(50) | 角色名称 |
| permissions | TEXT[] | 权限编码列表 (Array) |
