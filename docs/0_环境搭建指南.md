# 环境搭建指南

本文档旨在帮助开发人员快速搭建 [智途 (Zhitu)] 项目的本地开发环境。

## 1. 硬件与操作系统要求

- **操作系统**: Windows 10/11, macOS 12+, 或 Linux (Ubuntu 20.04+)
- **内存**: 建议 16GB 及以上 (微服务架构较为吃内存)
- **硬盘**: 建议预留 50GB 以上空间

## 2. 基础软件安装

请确保安装以下软件：

| 软件 | 版本要求 | 说明 | 下载地址 |
| :--- | :--- | :--- | :--- |
| **Docker** | 20.10+ | 容器化运行环境 | [Docker Desktop](https://www.docker.com/products/docker-desktop/) |
| **Git** | 2.30+ | 版本控制工具 | [Git SCM](https://git-scm.com/) |
| **JDK** | 17 (LTS) | 后端开发语言 | [Oracle JDK](https://www.oracle.com/java/technologies/downloads/#java17) 或 [OpenJDK](https://jdk.java.net/17/) |
| **Node.js** | 18 (LTS) | 前端运行时 | [Node.js](https://nodejs.org/) |
| **IntelliJ IDEA** | 2023.1+ | 推荐后端 IDE | [JetBrains](https://www.jetbrains.com/idea/) |
| **VS Code** | 最新版 | 推荐前端 IDE | [VS Code](https://code.visualstudio.com/) |

## 3. 中间件环境一键部署 (Docker Compose)

本项目提供了一键启动依赖服务的脚本。

### 3.1 启动服务

1. 进入项目根目录下的 `docker` 文件夹：
   ```bash
   cd docker
   ```
2. 启动所有服务：
   ```bash
   docker-compose up -d
   ```
3. 查看服务状态：
   ```bash
   docker-compose ps
   ```

### 3.2 服务清单与访问信息

| 服务 | 容器名 | 端口 | 账号/密码 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| **Nacos** | zhitu-nacos | 8848 | 无 (默认) | 注册中心与配置中心。控制台地址: `http://localhost:8848/nacos` |
| **PostgreSQL** | zhitu-postgres | 5432 | postgres / postgres_password | 业务主数据库。数据库名: `zhitu_cloud` |
| **Redis** | zhitu-redis | 6379 | (密码: `zhitu_redis_pwd`) | 缓存服务 |
| **MinIO** | zhitu-minio | 9000 (API) <br> 9001 (Console) | minioadmin / minioadmin | 对象存储。控制台地址: `http://localhost:9001` |

## 4. 后端开发环境配置 (Java/Spring Cloud)

1. **IDE 配置**:
   - 安装 **Lombok** 插件。
   - 建议安装 **Alibaba Cloud Toolkit** 插件。
2. **Maven 配置**:
   - 建议配置阿里云镜像源以加速依赖下载。
3. **导入项目**:
   - 打开 IntelliJ IDEA -> Open -> 选择项目根目录。
   - 等待 Maven 索引构建完成。

## 5. 前端开发环境配置 (React)

1. **包管理器**:
   - 推荐使用 `pnpm`：
     ```bash
     npm install -g pnpm
     ```
2. **Web 管理端**:
   - (待创建) 进入 `web-admin` 目录：
     ```bash
     cd web-admin
     pnpm install
     pnpm dev
     ```

## 6. 客户端开发环境 (Flutter) - 可选

如果需要开发移动端/桌面端应用：
1. 安装 **Flutter SDK** (3.10+)。
2. 运行 `flutter doctor` 检查环境。
3. 配置 Android Studio 或 VS Code 的 Flutter 插件。

## 7. 常见问题 (FAQ)

- **Q: Nacos 启动失败？**
  - A: 检查 8848 端口是否被占用。确保内存充足。
- **Q: 数据库连接不上？**
  - A: 确认 Docker 容器已运行，且端口 5432 未被本地其他 PG 实例占用。
