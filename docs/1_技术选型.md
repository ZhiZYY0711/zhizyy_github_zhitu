# 技术选型方案 (v2.0)

## 1. 前端技术栈 (Frontend)

本平台采用多端协同的策略，覆盖 Web 管理端、桌面端、移动端及微信小程序。

### 1.1 Web 管理端 (Admin & Portal)
面向高校管理员、企业 HR 和学生 PC 端用户，提供完整功能体验。
- **核心框架**: **React 18**
- **构建工具**: **Vite**
- **UI 组件库**: **shadcn/ui** (基于 Radix UI 和 Tailwind CSS)
  - *理由*: 现代化、可定制性强、源码级拷贝，符合现代审美。
- **样式系统**: **Tailwind CSS**
- **动画库**: **Animate.css** + **Framer Motion**
  - *理由*: 丰富的 CSS 动画预设 (Animate.css) 与复杂的交互动画 (Framer Motion)。
- **状态管理**: **Zustand** 或 **TanStack Query (React Query)**
- **路由管理**: **React Router 6**

### 1.2 跨平台客户端 (Desktop & Mobile App)
面向学生和企业导师的移动办公/学习场景，以及部分需要高性能交互的桌面场景。
- **核心框架**: **Flutter**
  - *理由*: Google 推出的高性能跨平台框架，一套代码同时构建 iOS, Android, Windows, macOS, Linux 应用。
- **状态管理**: **Provider** 或 **Riverpod**

### 1.3 微信小程序 (WeChat Mini Program)
面向学生的高频轻量级使用场景（如打卡、接收通知、查看简报）。
- **开发方式**: **原生开发 (Native)**
  - *理由*: 性能最佳，API 兼容性最好，适合利用微信生态能力（订阅消息、地理位置等）。
- **UI 库**: **WeUI** 或 **Vant Weapp**

## 2. 后端技术栈 (Backend)

采用微服务架构，以应对高并发、高可用及模块解耦的需求。

- **开发语言**: **Java (JDK 17)**
- **微服务框架**: **Spring Cloud Alibaba**
  - **注册中心/配置中心**: **Nacos**
  - **服务网关**: **Spring Cloud Gateway**
  - **服务调用**: **OpenFeign**
  - **负载均衡**: **Spring Cloud LoadBalancer**
  - **熔断限流**: **Sentinel**
- **应用框架**: **Spring Boot 3**
- **权限认证**: **Spring Authorization Server (OAuth2 / OIDC)** + **JWT**

## 3. 数据库与存储 (Database & Storage)

- **关系型数据库**: **PostgreSQL 15+**
  - *理由*: 
    - 强大的 JSONB 支持，适合存储非结构化的评价数据和复杂的简历结构。
    - 支持数组类型 (Array)，简化标签类数据的存储。
    - 优秀的并发控制 (MVCC) 和扩展性。
- **缓存数据库**: **Redis 7.0**
- **对象存储**: **MinIO** (自建) 或 **阿里云 OSS**

## 4. 部署与运维 (DevOps)

- **容器化**: **Docker** + **Docker Compose** (开发环境) / **Kubernetes (K8s)** (生产环境)
- **CI/CD**: **Jenkins** 或 **GitLab CI**
- **监控告警**: **Prometheus** + **Grafana**
