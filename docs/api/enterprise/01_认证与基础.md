# API 接口设计 - 企业端 (Enterprise Client) - 认证与基础

> **文档说明**: 企业端-认证与基础服务 API
> **服务**: `auth-service`, `user-service`
> **Base URL**: `/api/{service}/v1`

## 1. 认证服务 (Auth)

> **注意**: 所有端统一使用 `POST /api/auth/v1/login`，通过 `role` 字段区分端类型。
> 完整接口定义见 `student/01_认证与基础.md` 第 1 节，此处仅说明企业端特有的 Response 差异。

### 1.1 统一登录接口（企业端 Response）
- **URL**: `POST /api/auth/v1/login`
- **Request Body**:
  ```json
  {
    "role": "enterprise",
    "login_type": "account",
    "username": "hr_01",
    "password": "encrypted_xxx"
  }
  ```
- **Response**:
  ```json
  {
    "access_token": "...",
    "refresh_token": "...",
    "expires_in": 7200,
    "user_info": {
      "id": "20001",
      "name": "王HR",
      "role": "enterprise",
      "sub_role": "hr",
      "enterprise_id": "ent_001"
    }
  }
  ```
  > `sub_role` 枚举: `hr`(招聘专员) | `mentor`(企业导师) | `admin`(企业管理员)

### 1.2 刷新 Token / 登出
> 同 `student/01_认证与基础.md` 1.2、1.3 节，接口完全一致。

## 2. 企业基础信息 (Basic)

### 2.1 获取企业详情
- **URL**: `GET /api/user/v1/enterprise/profile`
- **Response**:
  ```json
  {
    "id": "ent_001",
    "name": "字节跳动",
    "logo": "https://...",
    "description": "...",
    "industry": "互联网",
    "scale": "10000人以上",
    "address": "北京市海淀区...",
    "verification_status": "verified"
  }
  ```

### 2.2 更新企业信息
- **URL**: `PUT /api/user/v1/enterprise/profile`
- **Request Body**:
  ```json
  {
    "description": "新的简介...",
    "logo": "new_url"
  }
  ```

### 2.3 员工管理（管理员）
- **URL**: `GET /api/user/v1/enterprise/staff`
- **Response**: 员工列表。

- **URL**: `POST /api/user/v1/enterprise/staff`
- **描述**: 添加新员工（如导师）。
- **Request Body**:
  ```json
  {
    "username": "mentor_01",
    "name": "李技术",
    "sub_role": "mentor",
    "email": "li@byte.com"
  }
  ```
