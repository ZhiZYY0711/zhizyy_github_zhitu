# API 接口设计 - 平台端 (Platform Client) - 认证与系统管理

> **文档说明**: 平台端-管理员登录与系统基础管理 API
> **服务**: `auth-service`, `system-service`
> **Base URL**: `/api/{service}/v1`

## 1. 管理员认证 (Auth)

> **注意**: 所有端统一使用 `POST /api/auth/v1/login`，通过 `role` 字段区分端类型。
> 完整接口定义见 `student/01_认证与基础.md` 第 1 节，此处仅说明平台端特有的 Request/Response 差异。

### 1.1 统一登录接口（平台端 Response）
- **URL**: `POST /api/auth/v1/login`
- **Request Body**:
  ```json
  {
    "role": "platform",
    "login_type": "account",
    "username": "admin",
    "password": "encrypted_xxx",
    "mfa_code": "123456"
  }
  ```
  > `mfa_code` 多因素认证码，平台管理员开启 MFA 后必填。
- **Response**:
  ```json
  {
    "access_token": "...",
    "refresh_token": "...",
    "expires_in": 7200,
    "user_info": {
      "id": "40001",
      "name": "超级管理员",
      "role": "platform",
      "permissions": ["system:all", "audit:view"]
    }
  }
  ```

### 1.2 刷新 Token / 登出
> 同 `student/01_认证与基础.md` 1.2、1.3 节，接口完全一致。

## 2. 基础主数据管理 (Master Data)

### 2.1 行业/技能标签管理
- **URL**: `POST /api/system/v1/tags`
- **描述**: 维护全平台通用的行业标签、技术栈标签。
- **Request Body**:
  ```json
  {
    "category": "tech_stack",
    "name": "Spring Cloud",
    "parent_id": "tag_java"
  }
  ```
  > `category` 枚举: `industry`(行业) | `tech_stack`(技术栈)

### 2.2 证书模板管理
- **URL**: `POST /api/system/v1/certificates/templates`
- **Request Body**:
  ```json
  {
    "name": "实训结业通用证书",
    "background_url": "http://...",
    "elements_layout": "{...}"
  }
  ```
