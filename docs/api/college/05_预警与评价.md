# API 接口设计 - 高校端 (College Client) - 预警与评价

> **文档说明**: 高校端-学生异常预警与综合评价 API
> **服务**: `portal-college-service`, `growth-service`
> **模块**: 1.3 高校管理驾驶舱 - 预警指挥室, 4.2 评价体系

## 1. 预警指挥室 (Warning System)

### 1.1 获取预警学生列表
- **URL**: `GET /api/portal-college/v1/warnings`
- **Query**: 
  - `level=high` (high红色/medium橙色/low黄色)
  - `type=missing_report` (missing_report周报缺失/attendance_abnormal考勤异常/unemployed未就业)
  - `status=unhandled` (unhandled未处理/handling处理中/resolved已解决)
- **Response**:
  ```json
  {
    "records": [
      {
        "id": "warn_001",
        "student_id": "stu_1001",
        "student_name": "赵六",
        "type": "attendance_abnormal",
        "level": "high",
        "description": "连续3天未在实习地打卡，且未提交请假条",
        "trigger_time": "2024-05-24 10:00:00",
        "status": "unhandled"
      }
    ]
  }
  ```

### 1.2 预警统计概览
- **URL**: `GET /api/portal-college/v1/warnings/stats`
- **Response**:
  ```json
  {
    "high_count": 5,
    "medium_count": 12,
    "low_count": 30,
    "type_distribution": {
      "missing_report": 20,
      "attendance": 15,
      "safety": 2
    }
  }
  ```

### 1.3 预警干预处理
- **URL**: `POST /api/portal-college/v1/warnings/{id}/intervene`
- **Request Body**:
  ```json
  {
    "method": "phone_call", // phone_call电话, face_to_face谈话, wechat微信
    "content": "已联系学生，因生病忘记打卡，已补假条",
    "status": "resolved", // resolved, handling
    "attachment_urls": [] // 通话截图等
  }
  ```

## 2. 综合评价管理

### 2.1 提交校方评价 (Process Evaluation)
- **URL**: `POST /api/growth/v1/evaluations/school`
- **描述**: 校内导师或辅导员对学生的实训/实习过程进行打分。
- **Request Body**:
  ```json
  {
    "student_id": "stu_1001",
    "project_id": "proj_001", // 或 internship_id
    "scores": {
      "norm_compliance": 90, // 规范性
      "document_quality": 85, // 文档质量
      "attitude": 95 // 态度
    },
    "comment": "表现良好，文档需加强"
  }
  ```

### 2.2 查看学生综合成绩单
- **URL**: `GET /api/growth/v1/evaluations/student/{studentId}`
- **Response**: 包含企业评价、校方评价、增值评价的综合得分详情。
