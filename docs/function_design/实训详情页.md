# 实训详情页

- [ ] 阶段1

### 阶段一：基础信息搭建（1-2 天）

**核心目标**：完成页面骨架搭建，展示实训核心静态信息，保证基础可读性

#### 具体任务：

1. 页面结构搭建
   - 确定页面整体布局（头部 / 主体 / 底部），划分信息模块区域（基础信息 / 内容介绍 / 师资展示）
   - 完成基础 UI 框架搭建（适配 PC 端，暂不考虑移动端）
2. 静态信息填充
   - 录入实训基础信息（名称、类型、周期、地点、人数、费用）
   - 上传实训内容介绍（学习目标、课程大纲、成果形式）
   - 完善师资力量展示（导师姓名、职称、简介，暂不展示联系方式）
3. 基础样式适配
   - 统一字体、颜色、间距等基础样式，保证信息排版清晰
   - 完成模块分隔（卡片、分割线），避免信息杂乱

#### 验收标准：

- 页面能正常加载，无样式错乱
- 所有核心静态信息展示完整、排版清晰
- 不同模块区分明确，用户能快速找到关键信息
- [ ] 阶段2

### 阶段二：核心交互功能开发（2-3 天）

**核心目标**：实现咨询、报名等核心操作功能，满足用户核心需求

#### 具体任务：

1. 联系方式功能开发
   - 为导师 / 咨询负责人添加联系方式展示（电话、微信、邮箱）
   - 实现「一键复制联系方式」功能，添加复制成功提示
   - 配置在线咨询入口（跳转企业微信 / 展示咨询二维码）
2. 报名功能开发
   - 制作醒目的报名按钮，区分「报名中 / 已截止」状态样式
   - 开发报名表单弹窗，包含基础信息填写项
   - 实现表单基础验证（必填项校验、手机号格式校验）
   - 配置表单提交成功 / 失败的反馈提示
3. 基础状态提示
   - 展示实训报名状态（未开始 / 报名中 / 已截止）
   - 配置报名条件前置提示（点击报名前弹窗提示报名要求）

#### 验收标准：

- 联系方式可正常复制，在线咨询入口跳转 / 展示正常
- 报名表单可正常填写、提交，验证规则生效
- 状态提示准确，用户操作反馈及时
- [ ] 阶段3

### 阶段三：体验优化与辅助功能（1-2 天）

**核心目标**：提升页面使用体验，补充辅助功能，降低用户操作成本

#### 具体任务：

1. 体验优化
   - 适配移动端布局，保证手机端浏览 / 操作正常
   - 为按钮、卡片添加 hover 交互效果，提升操作反馈
   - 开发 FAQ 模块，录入高频问题，实现展开 / 收起答案功能
2. 辅助功能开发
   - 实现「收藏实训」功能（本地缓存 / 关联用户账号）
   - 开发分享功能（支持复制链接 / 展示分享二维码）
   - 配置文件下载区（上传实训手册等文件，实现点击下载）
3. 细节优化
   - 优化表单填写体验（如输入框自动聚焦、字数提示）
   - 补充页面加载中 / 空数据的占位提示

#### 验收标准：

- 移动端 / PC 端展示、操作均正常
- 辅助功能（收藏、分享、下载）可正常使用
- FAQ 展开 / 收起交互流畅，用户操作无卡顿
- [ ] 阶段4

### 阶段四：数据闭环与功能完善（1-2 天）

**核心目标**：实现数据统计、状态同步，完善功能细节

#### 具体任务：

1. 数据统计功能
   - 同步「已报名人数」「浏览量」等实时数据
   - 实现个人报名状态查询（登录后显示「未报名 / 待审核 / 已通过」）
2. 功能闭环
   - 配置报名数据提交至后台，实现报名信息同步
   - 补充往届学员评价展示模块（录入评价内容、评分）
   - 开发已报名用户的反馈入口（提交实训建议）
3. 异常处理
   - 增加网络异常、接口请求失败的提示文案
   - 完善表单提交失败的兜底方案（如重新提交按钮）

#### 验收标准：

- 实时数据同步正常，个人报名状态展示准确
- 报名数据可正常提交至后台，反馈入口功能生效
- 异常场景有明确提示，用户操作无阻断
