## 五、版本管理规范

### 版本号规则
采用语义化版本号：`vMAJOR.MINOR.PATCH`

- **v1.0.0** - 初始版本
- **v1.1.0** - 新增功能（向后兼容）
- **v1.2.0** - 新增功能（向后兼容）
- **v1.3.0** - Bug修复（向后兼容）

### 文件夹命名
格式：`vX.Y.Z-description`

示例：
- `v1.0.0-init` - 初始化
- `v1.1.0-missing-api` - 缺失API
- `v1.2.0-talent-pool` - 人才库
- `v1.3.0-growth-fix` - 成长修复

### 必需文件
每个版本文件夹必须包含：
1. **README.md** - 版本说明文档
2. **upgrade.sql** - 升级脚本
3. **rollback.sql** - 回滚脚本（可选）
4. **verify.sql** - 验证脚本（可选）

## 六、使用指南

### 全新安装

#### Linux/Mac
```bash
cd database
chmod +x deploy.sh
./deploy.sh init
```

#### Windows
```cmd
cd database
deploy.bat init
```

### 升级数据库

#### Linux/Mac
```bash
./deploy.sh upgrade
```

#### Windows
```cmd
deploy.bat upgrade
```

### 验证数据库

#### Linux/Mac
```bash
./deploy.sh verify
```

#### Windows
```cmd
deploy.bat verify
```

### 手动部署

```bash
# 1. 初始化
cd v1.0.0-init
for file in 0*.sql; do
  psql -U zhitu_user -d zhitu_cloud -f "$file"
done
psql -U zhitu_user -d zhitu_cloud -f test_data.sql

# 2. 应用迁移
cd ../v1.1.0-missing-api
psql -U zhitu_user -d zhitu_cloud -f upgrade.sql

cd ../v1.2.0-talent-pool
psql -U zhitu_user -d zhitu_cloud -f upgrade.sql

cd ../v1.3.0-growth-fix
psql -U zhitu_user -d zhitu_cloud -f upgrade.sql
```
