# 环境配置说明

## 配置文件管理

本项目使用统一的环境变量配置文件来管理所有环境相关的配置，避免硬编码。

### 配置文件位置

- `.env` - 本地开发环境配置（不提交到 Git）
- `.env.example` - 配置模板（提交到 Git，供团队参考）

### 首次使用配置

1. 复制 `.env.example` 文件为 `.env`：
   ```bash
   cp .env.example .env
   ```

2. 根据实际环境修改 `.env` 中的配置值

### Nacos 配置

所有微服务的 `bootstrap.yml` 文件都使用以下环境变量：

- `NACOS_ADDR` - Nacos 服务器地址（默认：localhost:8848）
- `NACOS_NAMESPACE` - Nacos 命名空间 ID（当前：9f47ef46-0532-4144-9352-de979a04b191）

### 如何使用环境变量

#### 方式 1：IDE 配置（推荐）

在 IDEA 中配置环境变量：
1. Run -> Edit Configurations
2. 选择对应的启动配置
3. 在 Environment variables 中添加：
   ```
   NACOS_ADDR=localhost:8848;NACOS_NAMESPACE=9f47ef46-0532-4144-9352-de979a04b191
   ```

#### 方式 2：系统环境变量

在系统中设置环境变量（Windows）：
```cmd
set NACOS_ADDR=localhost:8848
set NACOS_NAMESPACE=9f47ef46-0532-4144-9352-de979a04b191
```

Linux/Mac：
```bash
export NACOS_ADDR=localhost:8848
export NACOS_NAMESPACE=9f47ef46-0532-4144-9352-de979a04b191
```

#### 方式 3：启动参数

使用 Maven 启动时传递参数：
```bash
mvn spring-boot:run -Dspring-boot.run.arguments="--NACOS_ADDR=localhost:8848 --NACOS_NAMESPACE=9f47ef46-0532-4144-9352-de979a04b191"
```

### 更新 Namespace ID

如果重新部署了 Nacos 容器，需要更新 namespace ID：

1. 登录 Nacos 控制台（http://localhost:8848/nacos）
2. 进入"命名空间"页面，查看或创建命名空间
3. 复制新的命名空间 ID
4. 更新 `.env` 文件中的 `NACOS_NAMESPACE` 值
5. 重启所有微服务

### 注意事项

- `.env` 文件已添加到 `.gitignore`，不会被提交到版本控制
- 团队成员需要根据 `.env.example` 创建自己的 `.env` 文件
- 生产环境应使用环境变量或配置中心，不要依赖 `.env` 文件
